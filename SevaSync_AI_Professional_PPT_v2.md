### 🟣 Slide 2: Problem Statement
*   **Crisis Information Overload:** During disasters, thousands of requests flood in, causing "Decision Paralysis" for responders.
*   **Lack of Prioritization:** Current systems treat all reports equally, leading to delays in life-threatening situations.
*   **Fragmented Coordination:** Communication between victims, NGOs, and volunteers is scattered across unofficial channels like social media.
*   **The Golden Hour:** The lack of real-time data visualization prevents responders from utilizing the "Golden Hour" effectively.

---

### 🟣 Slide 3: Our Solution - SevaSync AI
*   **Centralized AI Hub:** A unified platform that ingests community reports and processes them using an Intelligent Urgency Engine.
*   **Automated Triage:** Automatically ranks incidents based on severity, population impact, and resource requirements.
*   **Actionable Dashboard:** Provides NGOs with a "Command Center" view of the city’s status.
*   **Localized Response:** Connects help-seekers with the nearest available volunteers via geospatial mapping.

---

### 🟣 Slide 4: Key Features (Technical Depth)
*   **AI Urgency Scoring:** Uses Natural Language Processing (NLP) heuristics to analyze report descriptions and assign priority (1-10).
*   **Geospatial Visualization:** Interactive Leaflet maps with dynamic markers that change color based on urgency.
*   **Smart Volunteer Matching:** Algorithms that filter volunteers based on skill sets (Medical, Logistics, Rescue) and proximity.
*   **Dynamic Analytics Dashboard:** Live tracking of KPIs like 'Resolution Rate', 'High Urgency Hotspots', and 'Volunteer Deployment'.
*   **Automated Explanations:** The AI provides a reasoning for why a specific report was ranked as "High Priority".

---

### 🟣 Slide 5: How It Works (The Workflow)
1.  **Ingestion:** Users submit reports via a mobile-responsive web interface with location data.
2.  **Processing:** The FastAPI backend runs the AI engine to evaluate urgency and resource needs.
3.  **Prioritization:** Reports are stored in a structured database and pushed to the NGO dashboard in real-time.
4.  **Assignment:** NGOs assign tasks to specialized volunteers with one click.
5.  **Resolution:** Volunteers update the status on-field, closing the feedback loop and updating analytics.

---

### 🟣 Slide 6: System Architecture
*   **Frontend Layer:** Built with **React 18** and **Vite** for optimized performance and reactive state management.
*   **Backend Layer:** Powered by **FastAPI**, leveraging asynchronous processing for high-concurrency disaster scenarios.
*   **Logic Layer:** A modular **AI Urgency Engine** that processes unstructured text into structured priority scores.
*   **Database Layer:** **SQLAlchemy ORM** with **SQLite** for robust, relational data integrity and easy migration to PostgreSQL.
